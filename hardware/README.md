This folder contains the RTL and hardware setup notes for the FPGA implementation of the fixed-point split-variable DMM solver.

The design is intended to be rebuilt manually in Vivado. The repository does not include a complete Vivado project export.

Top-level custom IP wrapper: fixed_point_dmm_split_v_118_v1_0.sv
AXI4-Lite slave register module: fixed_point_dmm_split_v_118_v1_0_S00_AXI.sv
Main solver: dmm_fixed_pipelined_split_noILA.sv

## Vivado Block Design

The Vivado block design should include:

MicroBlaze processor
AXI interconnect
UARTLite
three AXI BRAM Controllers for clause storage
three external Block Memory Generator IPs for clause storage
custom DMM solver IP
required internal Block Memory Generator IPs used by the solver RTL

The three external clause BRAMs are outside of the custom solver IP. They are written by MicroBlaze before the solver starts and read by the custom IP during the solve.

## External Clause BRAM Configuration

Keep default properties generated by Vivado Block Design when adding AXI_BRAM_CTRL
Port A write width:       32 bits
Port A read width:        32 bits
Port B write width:       32 bits
Port B read width:        32 bits

Port A depth:             at least max number of clauses
Port B depth:             at least max number of clauses

## BRAM ADDRESS RANGE MUST BE LARGER THAN 4*DEPTH!!!

Examples:

BRAM depth = 2048 words    -> address range >= 8192 bytes
BRAM depth = 65536 words   -> address range >= 262144 bytes
BRAM depth = 131072 words  -> address range >= 524288 bytes

## Port A should be connected to an AXI BRAM Controller.

## Port B should be connected directly to the custom DMM solver IP.

The solver IP exposes one shared address and enable for all three clause BRAMs:

bram_addr_b
bram_en_b

Connect them as:

solver bram_addr_b  -> BRAM1 addrb
solver bram_addr_b  -> BRAM2 addrb
solver bram_addr_b  -> BRAM3 addrb

solver bram_en_b    -> BRAM1 enb
solver bram_en_b    -> BRAM2 enb
solver bram_en_b    -> BRAM3 enb

Connect the BRAM outputs back to the solver:

BRAM1 doutb -> solver bram1_dout_b
BRAM2 doutb -> solver bram2_dout_b
BRAM3 doutb -> solver bram3_dout_b

## Internal solver BRAM configuration

The solver core also instantiates several internal Block Memory Generator IPs directly inside the RTL.

blk_mem_gen_0  -> x_l memory
blk_mem_gen_1  -> x_s memory
blk_mem_gen_2  -> split-variable BRAM bank 1
blk_mem_gen_3  -> split-variable BRAM bank 2
blk_mem_gen_4  -> tag BRAMs

Internal requirements:

x_l memory: simple dual port RAM
    width == 64 bits
    depth == max number of clauses
    operating mode == no change
    fill remaining memory location: 1000000000000 (1 in Q16.48)

x_s memory: simple dual port RAM
    width == 64 bits
    depth == max number of clauses
    operating mode == no change
    fill remaining memory location: 800000000000 (1 in Q16.48)

split-variable BRAM banks: true dual port RAM
    width == 64 bits
    depth == max number of variables
    port A operating mode == write first
    port B operating mode == write first

tag BRAMs: simple dual port RAM
    width == 16 bits or enough to store the step tag
    depth == max number of variables
    operating mode == read first
    fill remaining memory location: FFFF (anything that is not 0)
